GIT and Subversion

Subversion:
+ позволяет держать в рабочей копии часть репозитория;
+ Не нужно явно указывать бинарный файл, или текстовый;
+ Есть атрибуты файлов и каталогов (через них, например можно сделать файл исполняемым для Linux из Windows);
+ Отслеживается работа с директориями и перемещением файлов;
+ Атомарные коммиты;
+ есть такая штука как: svn:externals. Она позволяет разделить проекты с общими модулями
+ Версии всех файлов имеют единую сквозную нумерацию.

- Для коммита необходимо подключиться к серверу.

Git:
+ распределенная система контроля версий. В случае сбоя можно взять локальный репозиторий без потери истории изменений;
+ быстрее, чем Subversion. При тестовом сравнении скорости я получил разницу более чем в 10 раз;
+ позволяет делать локальные коммиты без доступа к центральному серверу;
+ позволяет передать сделанные изменения по почте;
+ каждый работает со своим репозиторием, изменения из которого периодически переносятся в основной;
+ не засоряет каждую директорию своей служебной информацией;
+ есть полезные операции (git grep, git bisect, git archive), которые в svn делаются только сторонними утилитами;
