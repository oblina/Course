ПП:
+ хорошо подходит для  математических алгоритмов, в которых есть сложные вычисления без выделения дополнительных сущностей
+ лёгок для изучения : cпецифических понятий в процедурном программировании не так уж и много:(модуль, тип данных, функция, конструкция, переменная, константа)
- плохо расширяемый
- плохо читаемый
- повторение кода
- неструктуирован


ООП:
+ хорошо подходит для больших и часто изменяющихся проэктах
+ повторное использование
+ использовать рефакторинг относительно легко
+ расширяемость
+ ООП дает возможность создавать extensible systems
+ локализация кода и данных улучшает наглядность и удобство сопровождения программного обеспечения
+ ООП была разработана для повышения эффективности и облегчения процесса разработки
+ инкапсуляция информации защищает наиболее критичные данные от несанкционированного доступа
- сложность в проектирование классов
- изучение множество библиотек для работы
- необходимо понимать базовые концепции, такие как классы, наследование и динамическое связывание
